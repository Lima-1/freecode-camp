Why Flexbox

For a long time, the only reliable cross-browser compatible tools available for CSS layout were features like floats and position. These work, but in some ways ther're also limiting and frustrating.

The following simple layout designs are either difficult or impossible to achieve with such tools in any kind of convenient, flexible way:

  1. Vertically centering a block of content inside its parent.
  2. Making all the children of a container take up an equal amount of the available width/height, regardless of how much width/height is available.
Flexbox Model

When elements are laid out as flex items, they are laid out along two axes:

  1. The main axis is the axis running in the direction the flex items are laid out in (for example, as a row across the page, or a column down the page.) The start and end of the axis are called the main start and main end
  2. The cross axis is the axis running perpendicular to the direction the flex items are laid out in. The start and end of this axis are called the cross start and cross end.
  3. The parent element that haas dispay: flex set on it (the section in our example) is called the flex container.
Specifying Elements Layout

To start with, we need to select which elements are to be laid out as flexible boxes. To do this, we set a special value of display on the parent element of the elements you want to affect. In this case we want to lay out the article elemts, so we set this on the section:

                
                    section {
                        display: flex;
                    }

This causes the element to become a flex container and its children becomes flex items.

Columns or Rows

Flexbox provides a property called flex-direction that specifies the main axis runs(which direction the flexbox children are laid out in).

By default this is laid out in a row, which causes them to be laid in a row direction in the direction your browser's default language works in (left to right, in the case of English browser).

Try adding the following declaration to your section rule: 

                
                    flex-direction: column;

This will put the items back in a column layout, much like they were before we added CSS.

Wrapping

One issue that arises when you have a fixed width or height in your layout is that eventually your flexbox children will overflow the their container, breking the layout. Have a look at our flexbox0.html example and try viewing it live.

One way in which you can fix this is to add the followng declaration to your section rule:

                
                    flex-wrap: wrap;

Also, add the following declaration to your article rule:

                
                    flex: 200px;

there exist a shorthand for flex-direction and flex-wrap: flex-flow. Example:

                
                    flex-direction: row;
                    flex-wrap: wrap;

with

                
                    flex-flow: row wrap;
